# SQL Homework — Практика работы с базами данных

Проект содержит решения 13 SQL-задач по четырём базам данных: транспортные средства, автомобильные гонки, бронирование отелей и структура организации.

## Базы данных

### 1. `vehicles` — Транспортные средства
Хранит информацию об автомобилях, мотоциклах и велосипедах.

| Таблица    | Описание                              |
|------------|---------------------------------------|
| Vehicle    | Базовая таблица (производитель, модель, тип) |
| Car        | Автомобили (VIN, объём, мощность, цена, трансмиссия) |
| Motorcycle | Мотоциклы (VIN, объём, мощность, цена, тип) |
| Bicycle    | Велосипеды (серийный номер, передачи, цена, тип) |

### 2. `car_races` — Автомобильные гонки
Результаты гоночных заездов с классификацией автомобилей.

| Таблица | Описание                             |
|---------|--------------------------------------|
| Classes | Классы авто (тип, страна, параметры) |
| Cars    | Автомобили (название, класс, год)    |
| Races   | Гонки (название, дата)               |
| Results | Результаты (авто, гонка, позиция)    |

### 3. `hotel_booking` — Бронирование отелей
Система управления бронированием номеров в отелях.

| Таблица  | Описание                                   |
|----------|--------------------------------------------|
| Hotel    | Отели (название, расположение)             |
| Room     | Номера (тип, цена, вместимость)            |
| Customer | Клиенты (имя, email, телефон)              |
| Booking  | Бронирования (номер, клиент, даты заезда)  |

### 4. `organization` — Структура организации
Иерархическая структура сотрудников с проектами и задачами.

| Таблица     | Описание                                      |
|-------------|-----------------------------------------------|
| Departments | Отделы                                        |
| Roles       | Роли сотрудников                              |
| Employees   | Сотрудники (иерархия через ManagerID)         |
| Projects    | Проекты (привязаны к отделам)                 |
| Tasks       | Задачи (назначены сотрудникам, в рамках проекта) |

## Запуск проекта

### Требования
- [Docker](https://docs.docker.com/get-docker/) и Docker Compose

### Шаг 1 — Запуск контейнеров

```bash
docker compose up -d
```

При первом запуске Docker автоматически выполнит все скрипты из папки `init/postgres/` — базы данных создадутся и наполнятся тестовыми данными.

### Шаг 2 — Подключение к базам данных

**Через Adminer (веб-интерфейс):**

Откройте [http://localhost:8080](http://localhost:8080) в браузере.

| Параметр | PostgreSQL     |
|----------|----------------|
| Система  | PostgreSQL     |
| Сервер   | `postgres`     |
| Пользователь |`student`   |
| Пароль   | `student`      |
| База данных | `vehicles`, `car_races`, `hotel_booking`, `organization` |

**Через DBeaver / DataGrip / TablePlus:**

- PostgreSQL: `localhost:5432`, user: `student`, password: `student`

### Шаг 3 — Выполнение задач

Откройте нужный `.sql`-файл из соответствующей папки и выполните запрос в выбранной базе данных.

Пример для первой задачи по транспорту:
```bash

# PostgreSQL
psql -h 127.0.0.1 -p 5432 -U student -d vehicles -f vehicles/task1.sql
```

### Остановка контейнеров

```bash
docker compose down        # остановить (данные сохраняются)
docker compose down -v     # остановить и удалить данные
```

## Технологии

- **PostgreSQL 17** — альтернативная СУБД (скрипты совместимы)
- **Docker Compose** — оркестрация контейнеров
- **Adminer** — веб-интерфейс для управления БД

## Ключевые SQL-техники

| Техника | Где используется |
|---------|-----------------|
| `JOIN` (INNER, LEFT) | Все задачи |
| `GROUP BY` + агрегатные функции | Все задачи |
| `HAVING` | Фильтрация групп |
| `CTE` (`WITH`) | car_races, hotel_booking, organization |
| `WITH RECURSIVE` | organization (задачи 1–3) |
| `UNION ALL` | vehicles задача 2 |
| `GROUP_CONCAT` | hotel_booking, organization |
| `CASE WHEN` | hotel_booking задача 3 |
| Подзапросы | car_races задачи 2–5 |
